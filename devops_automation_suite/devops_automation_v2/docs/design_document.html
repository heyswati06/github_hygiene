DevOps Automation Suite
Architecture & Workflow

Git hygiene enforcement, auto-alerting, and weekly leaderboard reporting for 70 GitHub Enterprise teams. This document describes how the system's components interact, who the actors are, and how data flows from scan to alert.

Platform
GitHub Enterprise (on-prem)
Language
Python 3.11+
Teams
70 repos · 1 org
Schedule
Weekdays 9am + Friday 8am
Section 01

Component Overview

Six Python modules, one config file, one test suite. Each module has a single responsibility.

⚙️
settings.yaml
Configuration
Single source of truth. Defines GitHub URL, org name, team lead emails, all hygiene thresholds, SMTP settings, and cron schedules. No secrets stored here — tokens come from env vars.
🔌
github_client.py
GitHub API Layer
Thin REST client for GitHub Enterprise. Handles auth, automatic pagination (100 items/page), rate limit back-off with exponential retry, and timeout management.
requestsurllib3
🔍
hygiene_checker.py
Core Check Engine
Runs 6 check types per repo. Returns structured Violation objects with severity, detail, and metadata. Saves results to reports/latest_hygiene.json. Exit code 1 if critical violations found.
github_clientPyYAML
📬
alerter.py
Alert Dispatcher
Reads hygiene JSON, routes violations to the correct team lead email, applies 24h cooldown (SQLite-backed), builds styled HTML emails, sends via SMTP. Champion always CC'd.
sqlite3smtplib
📊
reporter.py
Leaderboard Generator
Collects release frequency and LTFD per repo, merges hygiene scores, computes composite rank, and renders a full HTML leaderboard. Run every Friday. Output archived by date.
github_clientPyYAML
🕐
scheduler.py
Orchestrator
Ties all components together. Runs hygiene+alert pipeline on weekdays at 9am, weekly report on Fridays. Can run as a persistent process or be triggered by Jenkins/cron.
schedulesubprocess

Section 05

All Hygiene Checks

Every check, its trigger condition, severity, score impact, and recommended action.

#Check NameTrigger ConditionSeverityScore ImpactRecommended Action
01 Stale Branch Branch last commit > 2 days ago (non-protected) WARNING
>6 days → CRITICAL		 −10 / −20 Merge or delete the branch. Use feature flags for incomplete work instead of long-lived branches.
02 Oversized PR PR total lines changed > 400 (additions + deletions) CRITICAL −20 Split into smaller PRs. Use feature flags to merge partial work safely. Refactor is not an excuse for a big PR.
03 Unreviewed PR (SLA) PR open for > 4 hours with zero reviews WARNING
>12 hours → CRITICAL		 −10 / −20 Assign a reviewer immediately. APPROVED, CHANGES_REQUESTED, and COMMENTED all satisfy the SLA.
04 Commit Message Format Any commit in a PR not matching type(scope): description WARNING −10 Rewrite commit messages. Types: feat|fix|chore|docs|refactor|test|ci. Description min 10 chars.
05 Missing Branch Protection main or master has no branch protection rules in GitHub CRITICAL −20 Enable branch protection: require PR reviews, require status checks to pass, disallow force-push.
06 Direct Push to Main PushEvent to main/master within last 7 days (not via PR) CRITICAL −20 Retroactively review the commits pushed. Enable branch protection to prevent future occurrences.

Score Deduction Summary

−20
Per CRITICAL violation
−10
Per WARNING violation
0
Floor (score never below 0)

Key design decision: The hygiene checker and alerter are intentionally decoupled via the JSON file. This means you can re-run the alerter on old results, run checks without alerting, or plug in a different alerter (Slack, Teams) without touching the checker.
